### 3. [branching] GitHub #24471: "Rewind history lost after compaction in sessions with many rewinds (branching tree)" · anthropics/claude-code · open

**https://github.com/anthropics/claude-code/issues/24471**

**Context (Claude Code — /rewind + compaction interaction):** After auto-compaction fires in a session that has been rewound multiple times, the entire rewind history collapses — only post-compaction messages remain visible despite a large multi-day conversation existing. The user loses their full branching tree, not just the pre-compaction portion. This is a distinct issue from #55347 (covered June 30 — /rewind promising a fork but not creating one); this one is about *compaction erasing* the branching history you did have. Not in any prior exclusion list; issue number is below the freshness threshold but the failure mode hasn't been reported in prior radars.

**Opportunity:** MED — the Ctrl-B branching angle is adjacent: ccfind's Ctrl-B forks from historical messages in *past sessions* (not just the current in-flight session), so even if the current session's rewind tree gets compacted away, a user can open the original session via ccfind and branch from any prior message that's still in the JSONL. Worth a short comment noting the workaround.

**Drafted reply (GitHub comment):**
> The compaction + rewind interaction is brutal because you lose the tree metadata even if the underlying messages survive. One escape hatch: if the original session's JSONL still exists on disk (pre-compaction), `ccfind` can open it and Ctrl-B lets you branch from any prior user message — forking into a new session with that message as the new root. It's not a substitute for native branch-tree persistence, but it's a recovery path when compaction has already fired. Disclosure: I built ccfind — https://github.com/coolcorexix/claude-grep
