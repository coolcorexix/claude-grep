### 1. [branching] GitHub #55347 — est. April–May 2026, actively linked in June discussions
**"/rewind says 'conversation will be forked' but never creates a resumable fork"**
[https://github.com/anthropics/claude-code/issues/55347](https://github.com/anthropics/claude-code/issues/55347) · anthropics/claude-code

**Context (Claude Code / branching):** The `/rewind` command displays the message "The conversation will be forked" but mutates the session in-place instead — after rewinding, the original session disappears from `/resume` and no fork is created. Users expecting a non-destructive branch end up with a mutated session and no way back. The bug is especially sharp because it breaks the one escape-hatch people rely on when a Claude Code session goes wrong. This issue appears in search results specifically when people look up `/rewind` + "fork" behavior, suggesting it keeps resurfacing in active troubleshooting. The issue pre-dates June 29's radar entries (issue number ~55k vs #72k for June 28 filings) but remains open.

**Opportunity:** MED — thread may be older but `/rewind`'s broken fork promise directly exposes demand for working conversation branching. Ctrl-B in `ccfind` is the reliable alternative: it reads the JSONL directly and opens a true fork via Claude Code's native `--fork-session`, leaving the original session intact. The drafted reply positions this as the workaround, not a competitive product.

**Drafted reply (GitHub comment):**
> The in-place mutation (instead of the promised fork) is the key pain here — `/rewind` asks you to trust that the original is preserved and then deletes it. As a workaround while this bug is open: [`ccfind`](https://github.com/coolcorexix/claude-grep) ships a Ctrl-B prompt-tree feature that forks from any past user message in any session (not just the current one). It calls Claude Code's `--fork-session` under the hood, so the fork shows up in `claude --resume` and the source session is untouched. `/btw`-compatible. `brew install coolcorexix/tap/claude-grep`. (Disclosure: I'm the author.)
