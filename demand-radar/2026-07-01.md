### 1. [search] HN: "Beware, Claude Code deletes >30 day old transcripts. Anthropic won't fix it"
**https://news.ycombinator.com/item?id=48732846** · Hacker News · est. late June 2026 (ID sequence: follows item ~48528779 which is the June WAYWO; co-occurs in search results with the June 30 TheRegister article)

**Context (Claude Code):** Thread prompted by The Register's June 30 article revealing that Claude Code silently purges `.jsonl` session transcripts older than 30 days by default, with no warning and no recovery path. Anthropic's on-record response was that it's documented and intentional ("a security measure"). Commenter `ojura` identified the root cause as mtime-based deletion rather than activity-based — meaning anything that touches file modification time (an update, a filesystem sync) can accelerate deletion even for recent sessions. Commenters in the thread are actively asking what tools or workarounds exist. The phrase "Anthropic won't fix it" in the title signals the community has moved past hoping for a platform fix and is now looking for third-party solutions.

**Opportunity:** HIGH — thread is live, participants are explicitly looking for solutions, and `ccfind` is a direct answer. The reply should lead with value (here's how to make what you still have searchable) rather than product pitch.

**Drafted reply (HN comment — no char limit):**
> If you still have sessions on disk, the best recovery step before hunting down what's left: `brew install coolcorexix/tap/claude-grep` (or `nix run github:coolcorexix/claude-grep`) → run `ccfind`. It full-text searches across all your Claude Code sessions from all project directories in one TUI, so you can find a conversation by what was *said* rather than by session ID or title. Works on Claude Code, OpenCode, and Hermes sessions in one view.
>
> Won't help if the files are already gone — but if they're still on disk, it surfaces them instantly. (Disclosure: I built this after losing my own session context one too many times.)
