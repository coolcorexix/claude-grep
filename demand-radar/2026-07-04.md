### 1. [search] GitHub #59248: "Silent retention cleanup deletes session transcripts with no warning, opt-in, or recovery"
**https://github.com/anthropics/claude-code/issues/59248** · anthropics/claude-code · filed late June 2026 · open

**Context (Claude Code):** The canonical community issue for the "30-day silent deletion" story that The Register covered June 30. Pain is threefold: (1) no install-time disclosure or first-run dialog; (2) deletion is keyed to `mtime` rather than actual last-activity timestamp — meaning anything that touches the file (a backup tool, a sync daemon, an ls -la) resets the mtime, causing *recent* sessions to look old and get swept; (3) Anthropic's public response ("it's documented") landed poorly — users expected a warning, not a post-hoc doc search. One commenter summarizes it cleanly: "The code and git history for a project remained after the wipe. The reasoning trail — design discussions, debugging context, analysis — is gone." That sentence is the exact pitch for content-search tooling: you can't rebuild the *why* from git.

**Opportunity:** HIGH — issue is active (filed within the last week), directly on-brand for ccfind, and the mtime-bug angle gives a genuine technical comment hook. The thread is discussing workarounds; ccfind belongs in that conversation.

**Drafted reply (GitHub comment):**
> The mtime angle makes this worse than a simple retention default: even sessions you *thought* you had aren't safe if anything touched the file. Two things that helped me: (1) bumping `cleanupPeriodDays` in `~/.claude/settings.json` to something large; (2) using [`ccfind`](https://github.com/coolcorexix/claude-grep) to do a content search across everything still on disk, so at least I can grep for *what was said* across sessions rather than relying on title-only `--resume`. The content-search angle also helps because you can find older sessions *before* the next cleanup sweep deletes them. Works across Claude Code, OpenCode, and Hermes. (Disclosure: I built ccfind.)

### 2. [search] GitHub #70052: "[BUG] Chat session history deleted — session files missing from .claude/projects/"
**https://github.com/anthropics/claude-code/issues/70052** · anthropics/claude-code · filed June 22, 2026 · open

**Context (Claude Code):** Distinct from the retention-cleanup issue: the reporter's session `.jsonl` files have disappeared from `~/.claude/projects/` entirely — not purged by the 30-day sweep but missing from disk, with no clear trigger. The missing-files variant is harder to explain away with "check the docs" and is drawing a different crowd of frustrated users who didn't touch their settings.

**Opportunity:** MED — the issue is 11 days old, still unfixed, and the "files just gone" framing is slightly different from the cleanup story. ccfind can help surface *what remains* (cross-project grep to find sessions that haven't been lost) and the Ctrl-B branch-from-history feature has a related pitch: fork sessions before they age out. Worth a brief comment.

**Drafted reply (GitHub comment):**
> If the files are still there but you can't find *which* session contained what you needed, [`ccfind`](https://github.com/coolcorexix/claude-grep) (`brew install coolcorexix/tap/claude-grep`) does full-text search across all JSONL sessions in `~/.claude/projects/` — tags results by project and date, lets you resume with one keypress. If the files are genuinely gone, that's a harder problem, but at minimum you can grep what still exists before the next sweep. (Disclosure: I built ccfind.)
