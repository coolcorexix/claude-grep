**Drafted reply (GitHub comment):**

It's not the full merge-back in Tier 2, but it does give you a named branch-from-message checkpoint today. Happy to share how the fork file format works if it's useful for this spec.

### 3. /rewind says "conversation will be forked" but never creates a resumable fork [branching]
**Link:** https://github.com/anthropics/claude-code/issues/55347 · **Platform:** GitHub (anthropics/claude-code) · **Date:** May 1, 2026

**Context (Claude Code):** Bug report with full JSONL analysis showing `/rewind` mutates the session in place (resets the branch pointer, no new JSONL file, no new session ID) despite the UI saying "the conversation will be forked." The original end-state disappears from `/resume`. 204 lines, all `isSidechain: false`. Related issues: #24471 (rewind history lost after compaction), #47987 (summarize-from-here drops pre-rewind messages). Marked `stale` but clearly unresolved.

**Opportunity:** MEDIUM. This is a credibility thread — the community knows forking is broken in the native tool. Showing that claude-grep's Ctrl-B produces a real, resumable fork file (not a mutated JSONL) is the differentiating proof point.

**Drafted reply (GitHub comment):**
> Confirming this is still broken in 2.1.x — analyzed the JSONL after a `/rewind` and saw zero sidechain entries. The mutation-in-place behavior also means you silently lose the abandoned branch. As a workaround until this is fixed: Ctrl-B in claude-grep (github.com/coolcorexix/claude-grep) creates an actual new session file branched from the selected message, separate JSONL, resumable via `claude --resume`. Not a substitute for a native fix but it sidesteps the data loss.
