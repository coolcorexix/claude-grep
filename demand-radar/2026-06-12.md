### 2. [branching] Issue #32631 · anthropics/claude-code — "Conversation Branching — full spec for fork, merge, and tree navigation"
- **Link**: https://github.com/anthropics/claude-code/issues/32631
- **Platform**: GitHub
- **Date**: ~March 2026 (outside 1-week window, but ongoing with active comments)
- **Context**: Claude Code. A community-written full spec proposing `/switch <name>` to navigate between branches and `/merge <name>` to pull conclusions back — explicitly framing it as "git branch + git merge for conversations." The author notes that `/fork` exists but the return path (navigation, merging) is entirely missing.
- **Opportunity**: HIGH — detailed, well-reasoned issue with upvotes; the maker has independently shipped the "fork from any past message" part this issue asks for, making a comment genuinely additive.

**Draft GitHub comment**:
> The fork-then-navigate problem is exactly what pushed me to build the Ctrl-B feature in [claude-grep](https://github.com/coolcorexix/claude-grep). What we shipped: Ctrl-B on any past user-message row forks the session at that exact point — it writes a new JSONL session file in `~/.claude`'s native fork format (including the `parentSessionId` back-reference), so the branch shows up in `claude --resume` like any other session. No `/merge` equivalent yet, but "checkpoint from any historical message" works today and is `/btw`-compatible.
>
> Happy to document the exact fork-file schema we reverse-engineered if it's useful for implementing this properly upstream. The format is simpler than it looks.
