### 1. [branching] GitHub Issue #55347 — /rewind promises fork but silently mutates in-place
**Link:** https://github.com/anthropics/claude-code/issues/55347  
**Platform:** GitHub · anthropics/claude-code  
**Date:** ~May 1, 2026 (opened)  
**Agent(s):** Claude Code  
**Opportunity:** High — active open bug, confirmed reproducible, directly in the branching UX gap claude-grep's Ctrl-B fills.

**Context:** Confirmed bug: `/rewind` tells users "The conversation will be forked," but it mutates the session in-place. After rewinding, the original session disappears from `/resume` and `claude --resume` — no fork is ever created. Related Issue #19451 (Jan 2026) adds nuance: `/rewind` creates in-file branches via `parentUuid` relationships inside a single session file, *not* separate session files, so they're invisible to the resume picker by design. Multiple reporters. The gap: there is no native way to get a named, resumable fork from a past message point.

**Drafted comment:**
> The in-file-branch vs. separate-session distinction is the core issue — `/rewind` forks the conversation tree internally but never surfaces it as a resumable session. I built a workaround for this in `ccfind` (claude-grep): hit Ctrl-B on any row in session history to fork from that exact past user message, writing a /btw-compatible branch that *does* show up in `claude --resume` as its own session. Not a fix for `/rewind`, but it's what I reach for when I want a named fork. (I'm the maker — github.com/coolcorexix/claude-grep)
