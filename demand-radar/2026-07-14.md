## Pain — Reddit / HN / GitHub

### 1. [search] HN: "Beware, Claude Code deletes >30 day old transcripts. Anthropic won't fix it"
- **Link:** https://news.ycombinator.com/item?id=48732846
- **Platform:** Hacker News
- **Approx. date:** ~late May / early June 2026 (corroborated by The Register covering the story June 30)
- **Context:** Claude Code purges `~/.claude/projects/*.jsonl` files older than 30 days on every startup. No disclosure, no opt-in. Users are losing months of reasoning context. Anthropic called it a documented security feature; the community called it silent data destruction. The Register picked it up June 30, keeping the thread alive and generating fresh traffic. GitHub issue #62476 (May 26) documents one user losing 14 sessions / 1,315 prompts in a single startup.
- **Opportunity:** HIGH — thread still circulating; entering with genuine help first is natural.

**Drafted reply (HN comment):**

> The 30-day wipe is the most painful half of this story; the other half is that even when sessions *aren't* deleted, `--resume` only matches titles. If you remember what you *discussed* but not what you named the session, you're stuck grepping JSONL by hand.
>
> I ran into this at 1,000+ sessions and built [claude-grep](https://github.com/coolcorexix/claude-grep) (`ccfind`) to fix it — full-text search across Claude Code, OpenCode, and Hermes Agent history from one TUI, one keypress to resume. Transparent: it's my own project. If your sessions survive the purge, this gets you back into them.
