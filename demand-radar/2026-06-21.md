### 3. [branching] `[FEATURE] Add option to rewind conversation without creating a fork` — GitHub Issue #9279 · anthropics/claude-code — Oct 2025
**Link:** https://github.com/anthropics/claude-code/issues/9279
**Platform:** GitHub Issues (anthropics/claude-code)
**Approx. date:** October 2025 (old but maps a real UX gap; closed as "not planned")
**Context:** Every use of `/rewind` adds a new fork, making `--resume` increasingly cluttered with dead-end branches. User wants a true "undo" option that discards the current branch in place rather than always forking. Pain compounds over long debugging sessions: attempt 1 fails → fork; attempt 2 fails → another fork; attempt 3 succeeds → you now have 4 conversations in `--resume` for one task. "Not planned" means it stays a third-party concern.
**Opportunity:** **Med** — old issue but unresolved UX pain is still present daily; brief reference-implementation comment is on-topic.

**Drafted comment:**
> The fork-clutter problem is real — every rewind adds a new orphaned branch to `--resume`. We approached this from the other direction in **claude-grep**: Ctrl-B lets you explicitly fork at any *past* user message (rather than at the rewind point), with a UI that lets you name the branch and see the tree. The no-op rewinding problem doesn't go away, but having intentional forks separate from accidental ones keeps `--resume` cleaner. Full disclosure, I'm the maker: https://github.com/coolcorexix/claude-grep
