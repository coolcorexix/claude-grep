### 1. `[branching]` — GitHub Issue #55347 · anthropics/claude-code — May 1, 2026

**[/rewind says "conversation will be forked" but never creates a resumable fork](https://github.com/anthropics/claude-code/issues/55347)**

- **Platform:** GitHub Issues (anthropics/claude-code)
- **Date:** May 1, 2026 (~6 weeks old; still open, labeled confirmed bug)
- **Context (Claude Code):** `/rewind` displays "The conversation will be forked" then mutates the session in-place. The original session disappears from `claude --resume`. No new session ID is created. The reporter expected a git-style branch to appear. This is the canonical open bug for the rewind-as-fork workflow.
- **Opportunity:** HIGH — the bug is the exact gap claude-grep's Ctrl-B fills from outside the Claude Code process.

**Drafted GitHub comment (value-first, maker disclosure at end):**
> Small workaround while this gets patched: `ccfind` (claude-grep) reads session transcripts directly from `~/.claude/projects/` via a separate read path and lets you hit Ctrl-B on any past user message to fork from that point. The fork is written in the native Claude Code format, so it shows up in `claude --resume` and is `/btw`-compatible. No dependency on the internal rewind buffer, so this particular bug doesn't affect it. Might be useful as an escape hatch. (Disclosure: I built it — [https://claude-grep.vercel.app](https://claude-grep.vercel.app) / `brew install coolcorexix/tap/claude-grep`.)
